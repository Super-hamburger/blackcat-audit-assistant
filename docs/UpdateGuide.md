# 更新发布与跨电脑升级规范

更新完功能后，先在本机测试版验证。只有明确确认“发布新版本”后，才上传 GitHub Release 并更新 GitHub Pages 的 `update_manifest.json`。

其他电脑优先使用一键更新流程：检查更新、下载并校验安装包、退出主程序、由独立脚本覆盖安装并重新启动。手动下载安装包仍作为备用流程。

## 默认规则

- 没有明确发布确认，只做本机测试版。
- 发布前必须先自测核心功能，不能只验证窗口能打开。
- 一键更新只在打包安装后的软件中执行，源码运行时不会覆盖项目文件。
- 一键更新需要远程清单同时提供 `download_url` 和 `package_sha256`。

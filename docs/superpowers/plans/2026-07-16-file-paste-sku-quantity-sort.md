# SKU 数量分组排序实施计划

**目标：** 一件代发表按“单件商品、同款多件商品、多款商品”三类订单依次处理；每一类内部继续按货架顺序排列。

**范围：** 只改临时测试项目中的文件粘贴转换逻辑；不改版本号、发布包或远程更新清单。

## 规则

1. 单个 SKU 且数量为 `*1`：第一组。
2. 单个 SKU 且数量大于 `*1`：第二组。
3. 多个 SKU：第三组。
4. SKU 或数量无法识别：排在最后，不影响原有的标红和异常提示。
5. 每组内部仍使用既有货架排序；货架排序相同则保持原表顺序。

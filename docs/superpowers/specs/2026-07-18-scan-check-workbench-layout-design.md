# 扫码验单任务驾驶舱布局设计

## 目标

让操作员在连续扫码时，只需在扫码框、整体进度和当前出库单号三个位置之间移动视线，即可判断当前单据是否已成功扫描。

## 布局

扫码页面采用“任务驾驶舱”层级：

1. 扫码输入卡片保留在最上方。
2. 整体进度条移动到扫码输入卡片下方，显示 `已匹配 X / Y（Z%）`；它不再位于单据明细卡片的底部。
3. 当前验单信息改为主视觉卡片，紧跟进度条。出库单号是最大文本，使用高对比度颜色和加粗字重；SKU、商品名、扫描时间和当前结果为次级信息。
4. 当前单据明细和最近扫描记录保留在下方，作为复核信息，而非扫码主焦点。

## 视觉状态

主视觉卡片根据现有扫码状态显示：

- 未开始/已选单：深色中性底色，出库单号保持高可读性。
- 匹配成功：绿色状态点与简短放行提示。
- 异常拦截：红色状态点和原因，继续维持无模态、不中断扫码的行为。
- 异常提示超时后：恢复中性待扫文案与视觉状态，不留下成功色的错误文字。

现有服务层匹配、进度计算、未匹配源数据导出和本机统计不改变。

## 实现边界

仅调整 `ui/main_window.py` 的扫码页组件层级、对象名/样式和渲染方法，以及相应的 UI 回归测试。扫描服务和 Excel 逻辑不改动。更新本地测试版变更日志，不发布版本或修改远程更新清单。

## 验证

新增测试确认：进度控件位于扫码输入区域之后、当前出库单号具有主视觉对象名/样式、不同扫码状态仍渲染正确。使用 offscreen PySide6 运行相关 UI、扫码和文件粘贴测试，并执行源码与 EXE 自测。

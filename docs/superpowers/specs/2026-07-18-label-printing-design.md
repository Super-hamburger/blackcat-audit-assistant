# 面单打印设计

## 范围

新增独立的“面单打印”功能入口，放在主界面导航中“扫码验单”上方。它不改变黑猫网站的上传和生成流程：操作员先照常用文件粘贴生成上传表、在黑猫网站生成所有面单 PDF，再将这些文件交给本功能重新排序并输出可直接打印的 PDF。

本功能仅处理一件代发来源的黑猫面单。现有“面单压缩”功能保持不变。

完整成品表常包含 `SKU×N`、多 SKU 和数量无法确认的订单。面单打印在重新输出 PDF 时只保留 `SKU×1` 订单的面单；上述其他订单的面单自动排除。黑猫网站生成的原始 PDF 不会被修改或删除。

## 输入

一次任务要求选择三类文件：

1. 原始一件代发表：提供“客户编号”、参考单号和货架。
2. 文件粘贴生成的完整成品表：提供转换后的收件人信息，并以第一列内部 ID 保留参考单号。
3. 黑猫网站生成的一个或多个 PDF：可同时选择宅急便和投函文件。

程序先以“参考单号”将原始表与完整成品表关联，得到每个成品订单的客户编号、货架和 SKU 分类；再把 PDF 每一页与成品订单关联：

- 宅急便优先用规范化后的收件电话匹配，必要时用邮编和收件人姓名复核。
- 投函用规范化后的收件邮编和收件人姓名匹配，必要时用收件地址复核。

电话、邮编、姓名和地址在匹配前会去除空格、全角空格、连字符和电话/邮编格式差异。每页必须恰好对应一条订单。

SKU 分类与文件粘贴现有规则一致：仅一个 SKU 且数量恰为 1 的订单为 `SKU×1`；单 SKU 数量大于 1、两个或以上 SKU、SKU/数量无法确认的订单均为排除对象。程序先完成全部页面的匹配，再排除非 `SKU×1` 页面，因此不会把不属于源表的页面误认为可忽略内容。

## 排序和打印选择

客户编号只用于决定“按客户逐个打印”时的分组；所有输出的页序都以货架为主排序，以便拣货从头到尾不回头。货架使用现有的自然排序规则（数字段按数值比较，字母货架随后，无效或空货架最后）；货架相同则保持原 PDF 的原始页面顺序。

页面提供两个选择：

| 选择 | 选项 | 输出 |
| --- | --- | --- |
| 打印范围 | 全部客户一次打印 | 生成覆盖全部订单的 PDF。 |
| 打印范围 | 按客户逐个打印 | 以客户编号升序生成每个客户一份或两份 PDF。 |
| 面单类型 | 合并 | 宅急便和投函混合后统一按货架排序。 |
| 面单类型 | 分开 | 投函和宅急便分别按货架排序；投函文件排在前。 |

因此每次运行的输出如下：

- 全部客户 + 合并：`全部客户_合并_货架排序.pdf`。
- 全部客户 + 分开：`全部客户_投函_货架排序.pdf`、`全部客户_宅急便_货架排序.pdf`。
- 按客户 + 合并：`客户12027_合并_货架排序.pdf` 等。
- 按客户 + 分开：`客户12027_投函_货架排序.pdf`、`客户12027_宅急便_货架排序.pdf` 等；没有对应类型面单时不生成空文件。

页面提供输出路径选择；用户可指定保存位置，未指定时使用应用默认输出目录。程序在所选路径下按本次任务创建独立输出文件夹，完成后自动打开。结果区显示总页数、已匹配页数、已输出的 `SKU×1` 页数、已排除的 `SKU×N和多SKU` 页数、客户数、按类型的页数和全部输出文件名。若全部页面都被排除，不生成空 PDF，并明确提示没有可打印的 `SKU×1` 面单。

## 页面与模块边界

业务逻辑放在新的 `modules/label_printing/` 模块：负责 Excel 订单映射、PDF 页面文本读取、面单类型识别、匹配、稳定货架排序和 PDF 合并写入。主窗口只负责文件选择、两个选项、进度显示和结果展示。

导航项文字为“面单打印”，位置在“扫码验单”之前。页面需要三个文件选择区、输出路径选择、打印范围选择、面单类型选择、开始按钮、进度区和结果日志。

## 异常处理

任何一个 PDF 页面没有匹配、匹配到多个订单、重复导入同一张面单、来源表缺少必要字段，任务立即停止，不输出部分客户 PDF。结果目录仅保留一份异常报告，列出 PDF 名称、页码、识别的面单类型、候选订单和失败原因，供人工修正输入后重试。正常排除的 `SKU×N和多SKU` 页面不属于异常，结果区只汇总其数量。

## 验证

测试覆盖：

1. 原始表与完整成品表的参考单号映射，以及客户编号和货架提取。
2. 宅急便电话匹配、投函邮编加姓名匹配和格式规范化。
3. 全部客户/按客户，以及合并/分开四种输出组合的文件名、页数、类型顺序和货架顺序。
4. 同货架保持输入 PDF 页序；空类型不生成空 PDF。
5. 仅 `SKU×1` 页面进入输出；`SKU×N`、多 SKU 和数量无法确认页面被排除但不修改原 PDF。
6. 未匹配、歧义、重复页面和缺少字段时没有部分 PDF 输出，并生成异常报告。
7. 用用户提供的 102 条样本验证：两份 PDF 共 102 页全部唯一匹配；客户 12027、12028、12029 分别为 22、49、31 页，并验证非 `SKU×1` 页面不会进入打印结果。

完成后运行模块测试、静态检查、JSON 校验和源码自测，并按项目规则更新本地变更日志。

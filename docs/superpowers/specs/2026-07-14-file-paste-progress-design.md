# 文件粘贴生成进度设计说明

## 目标

文件粘贴生成成品表时，主窗口不再因 Excel 转换而无响应。用户会看到明确的生成进度动画，且动画从开始显示到结束至少持续 2 秒。

## 交互规则

- 用户点击“生成上传表”后，先完成文件路径校验，再显示“正在生成上传表”的模态进度窗口。
- 进度条从 0 向 95 平滑推进；后台任务未完成时，进度条最多停在 95，不提前提示成功。
- 后台任务完成后，如已显示不足 2 秒，则继续动画直到满 2 秒；随后进度条到 100，关闭进度窗口，并显示原有成功或失败提示。
- 生成期间禁用“生成上传表”按钮，避免同一文件被重复提交。
- 转换失败也遵守最少 2 秒展示规则，之后恢复按钮并显示错误提示。

## 实现范围

- 新增后台 `QThread` 工作对象执行现有 `file_paste` 模块，不修改 Excel 表识别、地址拆分、SKU、货架或 B 列发货方式逻辑。
- 主窗口使用 `QTimer` 驱动进度动画，并使用单调时间计算最少展示剩余时间。
- 原有生成成功日志、处理记录、自动打开 Excel 和完成提示保持不变，只在后台任务成功后执行。

## 验证要求

- 最少展示时间计算在已用时不足 2 秒时返回正确剩余毫秒数，超过 2 秒时返回 0。
- 生成任务在后台工作对象中调用模块注册表；成功和失败均通过信号返回主线程。
- 文件粘贴的现有转换测试仍全部通过。

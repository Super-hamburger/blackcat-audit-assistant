# 文件粘贴分组拆分输出功能说明

## 目标

文件粘贴生成黑猫上传表时，按商品组成拆分为多个独立 Excel 文件，方便仓库按“单 SKU”和“多 SKU”分别处理，并避免单个上传表数据过多。

## 分类与排序

1. 单 SKU：一单只有一个 SKU 的数据；`SKU*1` 与 `SKU*N` 放在同一类中。类内保持现有顺序：`SKU*1` 优先，其后为 `SKU*N`，并继续按货架顺序排列。
2. 多 SKU：一单包含多个 SKU 的数据，始终单独输出，绝不与单 SKU 混在同一份成品表。
3. 数量格式不能识别的数据不丢弃，仍按现有规则标红；按 SKU 个数归类：单 SKU 进入单 SKU 文件的末尾，多 SKU 或 SKU 为空进入多 SKU 文件的末尾。

## 拆分规则

- 每个独立 Excel 最多写入 999 条数据，预留表头行。
- 先完成分类，再在各类别内部按 999 条依次拆分；不为了凑到 999 条而把两类数据混在一起。
- 示例一：共 1300 条，单 SKU 900 条、多 SKU 400 条，输出两份：单 SKU 900 条和多 SKU 400 条。
- 示例二：共 500 条，单 SKU 300 条、多 SKU 200 条，仍输出两份。
- 示例三：单 SKU 1200 条、多 SKU 400 条，输出三份：单 SKU 第 1 份 999 条、单 SKU 第 2 份 201 条、多 SKU 第 1 份 400 条。

## 输出目录与命名

- 每次生成创建一个单独文件夹：`黑猫上传表_YYYYMMDD_HHMMSS`。
- 该次生成的所有 Excel 都放在此文件夹内。
- 文件名使用固定排序和数量信息：
  - `01_单SKU_第1份_900条.xlsx`
  - `02_单SKU_第2份_201条.xlsx`
  - `03_多SKU_第1份_400条.xlsx`
- 若某分类没有数据，不生成该分类的空文件。
- 完成后自动打开输出文件夹；界面结果显示该文件夹路径和生成文件数量。

## 保持不变的行为

- 使用既有黑猫上传模板、98 列表头和字段映射。
- 保持地址拆分、红黄标记、运输方式写入与 SKU 文本处理。
- 保持原有单 SKU 的货架排序优先级。

## 验收标准

1. 单 SKU 和多 SKU 的输出文件互不混合。
2. 每份 Excel 的数据行数不超过 999 条。
3. 900 条单 SKU与 400 条多 SKU输出为两份文件，不因总数 1300 而拆成 999 与 301。
4. 单 SKU 1200 条正确拆成 999 与 201 条。
5. 文件都位于同一新建文件夹，名称带分类、份数与条数。
6. 原有排序、字段和异常标红回归测试通过。

# 扫码枪兼容输入设计

## 目标

让扫码验单兼容不同扫码枪的结束方式：发送回车、发送 Tab，或不发送结束键。出库单号和 SKU 都必须按一次扫码只提交一次的规则处理，避免其他电脑无法选中出库单。

## 输入规则

1. 扫码枪发送回车时，立即提交当前输入内容。
2. 扫码枪发送 Tab 时，立即提交当前输入内容，并阻止焦点跳到下一个控件。
3. 没有结束键的扫码枪：快速连续输入完成后，停顿 150 毫秒自动提交。
4. 自动提交仅对至少 4 个字符、并在 1 秒内连续输入完成的内容生效；慢速人工输入不会被自动提交，仍可按回车确认。
5. 任一提交入口都会先停止等待计时、读取并清空输入框，再调用现有扫码验单服务，确保同一条码只处理一次。

## 实现范围

- 在 ui/main_window.py 为扫码输入框增加单次触发计时器和输入速度状态。
- 统一回车、Tab 和自动计时器的提交入口，复用既有 handle_scan_input 的扫码服务、提示音和异常拦截流程。
- Tab 通过输入框事件过滤器拦截，保持扫码输入框焦点。
- 不改变 Excel 读取、出库单/SKU 匹配规则、多电脑统计规则或异常导出。

## 错误处理与测试

- 空输入不提交；扫码结束后输入框始终重新获得焦点。
- 自动提交未达到快速输入条件时不触发，用户可继续编辑或按回车。
- 新增界面层的测试，覆盖回车、Tab、快速无结束键输入、慢速输入不自动提交，以及一次输入只调用一次验单服务。
- 保留现有扫码导出测试，确保导出逻辑未受影响。

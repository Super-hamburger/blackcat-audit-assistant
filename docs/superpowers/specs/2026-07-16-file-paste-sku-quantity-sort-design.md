# 成品表 SKU 数量分组排序设计

## 目标

在“一件代发表格”转换为成品表时，按订单 SKU 明细的结构排序，以便拣货：

1. 单个 SKU 且数量为 `*1` 的订单排在最上方。
2. 单个 SKU 且数量大于 `*1` 的订单排在中间。
3. 含多个 SKU 的订单排在最下方。

同一分组内保持当前的货架排序。黑猫新版表格的转换顺序不变。

## 方案

在 `modules/file_paste/converter.py` 中新增一个只负责计算分组优先级的函数。该函数复用现有的 SKU 分隔和数量解析规则：

- 一个 SKU、可识别数量为 `1`：优先级 0。
- 一个 SKU、可识别数量大于 `1`：优先级 1。
- 两个及以上 SKU：优先级 2。
- SKU 或数量无法解析：优先级 3，排在最后，避免把异常数据误当成可拣货订单。

一件代发表格的最终排序键为“SKU 分组优先级 + 现有货架排序键”。因此仅改变三类订单之间的先后次序；每一类订单内部仍按当前货架规则排列，且货架排序相同的订单保持源表相对顺序。

## 兼容性与错误处理

- 导出的 SKU 明细、数量标红、地址处理及其他字段映射保持不变。
- 多 SKU 订单即使每项都是 `*1`，仍归入最后一组。
- 数量不明确或格式无法识别的订单继续走现有错误提示和标红逻辑，不丢弃数据。

## 测试

在现有转换器测试中加入端到端场景：混合单 SKU `*1`、单 SKU `*3`、多 SKU 的输入行，断言导出订单顺序为三组顺序，并断言每组内部仍遵循现有货架排序。原有测试将同步调整为新规则。

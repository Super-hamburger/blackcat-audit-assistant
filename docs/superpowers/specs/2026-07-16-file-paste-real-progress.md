# 文件粘贴真实进度与可控任务功能说明

## 背景

文件粘贴功能处理约 6000 条数据时速度明显变慢。当前弹窗使用计时器模拟进度，短时间内会显示为 95%，随后长时间停住，不能反映真实处理状态；用户也无法暂停或结束任务。

## 目标

1. 用实际工作阶段和实际行数反馈处理进度，不再使用按时间模拟的 95% 进度。
2. 降低逐行复制模板样式的重复开销，改善大表生成速度，并保持现有模板样式与导出数据不变。
3. 在任务弹窗中提供暂停、继续和结束任务能力。
4. 用户结束任务时，丢弃未完成结果，绝不留下半成品上传表。

## 范围与行为

### 真实进度

- 打开 Excel、排序、保存属于无法可靠按单元格计算百分比的阶段，界面显示明确阶段文字和忙碌进度条，不显示虚假的 95%。
- 读取数据时按“已读取行数 / 源表数据行数”更新。
- 写入上传表时按“已处理订单数 / 有效订单总数”更新。
- 界面展示当前阶段、已处理数量与总数量；写入完成后才进入保存阶段。

### 暂停与结束

- 点击“暂停”后，当前行处理完成即暂停；按钮文字改为“继续”。
- 点击“继续”后从当前行继续，不重复已完成行。
- 点击“结束任务”后，当前行处理完成即退出，显示“正在结束”。
- 结束、异常或保存失败时关闭工作簿并清理本次尚未完成的输出文件；成功保存后才将最终文件暴露给用户。
- Excel 文件正在打开或正在最终保存时，第三方库本身不能安全中断；界面仍可登记结束请求，并在该阶段结束后立即清理退出。

### 性能

- 源表以只读模式读取，避免为仅需数据的源文件加载格式对象。
- 复制模板行时复用不可变的样式定义，去除每个目标单元格不必要的二次复制；写入后仍可针对异常单元格单独标红。
- 不改变既有的 SKU 分组、货架排序、地址拆分、字段映射和异常标记规则。

## 技术设计

- `modules/file_paste/converter.py` 新增线程安全的 `ConversionControl`（暂停、继续、结束）及 `ConversionCancelled`；`UploadConverter.convert` 接收可选的 `progress_callback` 与 `control`。
- 转换器在打开、读取、排序、写入、保存、完成阶段发送字典进度事件。读取与写入事件必须带 `current`、`total`；无法量化的阶段以 `indeterminate=True` 标记。
- `modules/file_paste/module.py` 将界面上下文中的控制对象与回调传递给转换器。
- `ui/main_window.py` 的后台线程转发进度信号；文件粘贴对话框按进度事件更新，提供暂停/继续和结束按钮，不再启动模拟计时器或最短展示延迟。

## 验收标准

1. 转换器为读取、写入、保存等阶段发出可验证的进度事件，读取和写入的总数与实际数据一致。
2. 暂停后转换不会继续处理下一行；继续后可以正常完成。
3. 结束后抛出可识别的取消结果，输出目录没有新生成的 `.xlsx` 半成品。
4. 成功转换保持现有字段、排序和异常标记结果；模板行样式仍正确。
5. 界面弹窗显示真实阶段文字和行数，支持暂停/继续/结束，完成时只显示完成或失败/已结束。
6. 现有自动化测试与新增回归测试均通过；使用 6000 行测试表验证任务能在后台运行、进度单调递增，且不再出现模拟的 95%。

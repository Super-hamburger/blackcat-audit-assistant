# 文件粘贴与扫码验单优化设计

## 范围

本次只优化文件粘贴和扫码验单，不发布新版本、不修改远程更新清单或发布包。

## 文件粘贴

现有成品表按“单 SKU”和“多 SKU”分组。改为生成以下三类成品表，并继续按每份最多 999 条拆分：

1. **SKU*1**：仅一个 SKU，且解析后的数量恰为 1。
2. **SKU*N和多SKU**：仅一个 SKU 且数量大于 1，以及含两个或以上 SKU 的订单。
3. **完整成品表**：全部有效订单，保持现有排序结果，供用户需要统一导入时使用。

每个输出文件使用明确中文标签和顺序编号。转换返回的文件列表、数量和进度消息必须包含完整成品表。数量或 SKU 无法解析的订单保持现有处理方式：仍进入输出，且保留数量问题标记；此类单 SKU 订单归入“SKU*N和多SKU”组，避免误归类到 SKU*1。

## 扫码验单：进度与视觉反馈

扫码页继续只统计本机导入批次。整体进度定义为：

`已成功匹配 SKU 数量 / 可验单 SKU 总数量`

每次成功扫描一个 SKU，分子增加一；选中出库单、失败扫码或暂停均不改变进度。进度条显示百分比，并加上清晰的“已匹配 X / Y”文字。

匹配成功使用醒目的绿色状态区；异常拦截使用醒目的红色状态区、加粗原因文字和短暂闪烁。两种状态均更新当前订单、SKU 和商品信息。

## 扫码验单：无鼠标异常处理

移除所有异常扫码触发的模态弹窗。异常时仅展示非阻塞红色视觉提示、播放现有报警音，并立即把输入焦点还给扫码框。操作员可直接进行下一次扫描；下一次结果会覆盖当前提示，提示也会自动恢复为正常待扫状态。不会要求鼠标点击或键盘确认。

## 扫码验单：移除异常记录

彻底删除异常记录能力：

- 不再在服务中保存异常扫码、来源行或异常导出数据。
- 移除异常记录表格和“导出异常记录”按钮与方法。
- 删除相关导出逻辑和测试。

“导出未匹配源数据”保留：它根据尚未成功匹配的源表行生成文件，不使用异常扫码枪内容，且与异常记录完全分离。

## 实施边界与验证

业务分组和匹配状态保留在模块层；主窗口只负责显示和事件处理。新增/修改单元测试将覆盖三类表的分组与完整表、进度分子分母、异常不持久化，以及无模态异常提示。完成后运行相关测试、静态检查、JSON 校验及源码自测；在交付前更新本地变更日志。

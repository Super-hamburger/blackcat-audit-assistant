# 扫码异常提示与英文输入设计

## 目标

改善扫码验单现场体验：扫码异常必定有清晰警示音；扫码输入框获得焦点时默认偏好英文小写输入，不影响扫码枪数据。

## 扫码异常声音

- 将已确认的双提示音资源保存为 assets/sounds/scan_error.wav。
- 只有扫码验单产生 result 为 block 的异常时，播放该专用声音。
- 专用扫码异常声音不受统一 UI 音效开关影响；点击、导入和其他页面的声音仍遵循原开关。
- 保持原有异常弹窗、异常记录、统计和拦截逻辑不变。

## 扫码输入规则

- 扫码输入框设置仅拉丁字符和偏好小写的 Qt 输入法提示。
- 输入框获得焦点时，Windows 尝试激活英文键盘布局；系统不支持时静默退回 Qt 输入法提示。
- 扫码枪的回车、Tab 和无结束键自动提交规则保持不变。

## 测试

- 验证扫码异常使用专用播放入口，即使统一 UI 音效关闭也会调用声音引擎。
- 验证扫码输入框设置英文小写输入法提示。
- 验证 scan_error.wav 是有效的 WAV 音频资源。
- 运行现有扫码输入兼容、扫码导出和全量回归测试。

# B 列发货方式设计说明

## 目标

生成黑猫上传成品表时，固定在 B 列（`送り状種類`）写入正确的发货方式代码：

- `A`：黑猫投函。
- `0`：黑猫宅急便。

## 数据来源

- 一件代发表：读取可选列“运输方式”。内容包含“投函”时写入 `A`，包含“宅急便”时写入 `0`。
- 黑猫新版表：读取可选列“备注”。内容包含“黑猫投函”时写入 `A`，包含“黑猫宅急便”时写入 `0`。
- 两种来源的识别所需列均为可选列，不改变原始表类型的识别条件。

## 异常处理

- 来源字段为空、未包含可识别关键词，或同时包含“投函”和“宅急便”时，B 列保持为空。
- 上述异常的 B 列使用现有红色标记，提醒人工确认；订单仍会生成，不中断整个文件转换。
- 其他成品表列、地址拆分、货架排序、SKU 分配和模板表头位置均保持不变。

## 验证要求

- 一件代发表的“运输方式”为“宅急便”时，B2 为文本 `0`。
- 黑猫新版表备注中含“黑猫投函”时，B2 为文本 `A`。
- 黑猫新版表备注为空时，B2 为空且使用红色标记。
